VMware 17.6.4版本安装与Rocky Linux 10.1虚拟机部署

xiao yu2026-03-272026-03-31

前言

本文主要记录在 Windows 11 环境下，安装 VMware Workstation Pro 并创建 Rocky Linux 虚拟机的完整过程，用于后续 Linux 学习与运维实验环境搭建。

环境介绍

操作系统：Windows11专业版
VMware版本：17.6.4

下载VMware Workstation Pro

在浏览器输入vmware.com地址，或者点击我这个超链接 [VMware官网网站]

点击Products（产品）

点击SEE DESKTOP HYPERVISORS（查看桌面虚拟机管理程序）

点击DOWNLOAD FUSION OR WORKSTATION（下载Fusion或工作站），会跳转到BROADCOM（博通公司）界面

点击LOGIN-REGISER，使用邮箱注册来进行登录（有账号的可以跳过这步）

输入邮箱用于接受验效验码并填写图片的验证码

查看自己的邮箱里邮件的验效验码并复制进去

我这里是用的谷歌邮箱，QQ邮箱验证码很慢甚至一直等不到

填写创建账号的信息
名字可以随便填，密码需要数字、大写字母、小写字母、特殊符号组成；

直接点击 I’ll do it later（我稍后会做），到此注册成功

再回到登录页面进行登录

输入密码进入页面

登录后进入如下界面，然后点My Downloads （我的下载）- HERE

搜Workstation Pro然后左键点击

是Windows系统就选Windows版，是Linux系统就选Linux版
25H2版本是25年发布的，对比Pro 17.0如下

对比项 25H2 17.x
版本命名 年份制（25H2） 传统版本号（17.6.x）
发布时间 2025年 2022~2024年
虚拟硬件版本 v22 v19 ~ v21
单虚拟机内存上限 最大支持24TB 最大支持128GB
CPU 支持 支持最新 CPU（如新一代 Intel） 支持较旧 CPU
USB 支持 USB 3.2 USB 3.0 / 3.1
Hyper-V 共存 更稳定（Win11优化） 一般
新功能 dictTool、增强检测等 无

选择17.6.4版本
这里说明一下：
VMware从17.6开始提供“免费个人使用（无需密钥）”模式，17.5以下必须要输入许可证密钥

要先点击蓝色字，然后才能勾选上面的方框，同意条款和条件，确认下载的软件信息之后点击下载

如果是第一次下载，需要提交额外信息，提交之后才能返回到刚上面的页面进行下载
在下载此文件之前，需要进行额外验证，点击yes

提交额外信息

提交完额外信息之后，会跳转到刚刚的下载页面
下载完成，可以把文件放在某个文件夹进行安装

为了方便大家，在这里我给大家分享VMware17.6.4版本安装包的网盘链接
我用夸克网盘分享了「VMware安装包」，点击链接即可保存。打开「夸克APP」，无需下载在线播放视频，畅享原画5倍速，支持电视投屏。链接：https://pan.quark.cn/s/ade84b482e62 提取码：SBAY

安装VMware Workstation Pro

右键点击选择“以管理员身份运行”下载好的exe文件

弹出“用户账户控制”提示，点击“是”来允许此应用对你的设备进行更改

点击下一步

点击“我接受许可协议中的条款”-下一步

点击更改目录，自定义一个位置就可以，然后点击下一步

更改完路径会弹出一个警告（提示更改安装路径可能不安全），点击确定就行

有些人可能会遇到兼容性问题：安装程序检测到主机启用了Hyper-V或Device/Credential Guard

是因为你的Windows开启了Hyper-V或者相关的安全功能（如内核隔离/防护），这会和 VMware 的运行机制产生冲突
Hyper-V是微软开发的一种虚拟化技术作用就是在你的物理电脑（宿主机）上，划分出独立的小房间，让你能同时运行多个不同的操作系统（虚拟机），比如在 Windows 11 里再跑一个 Linux 或另一个 Windows
两种方法解决：

  • 直接勾选底部的“自动安装 Windows Hypervisor Platform (WHP)”，然后点击“下一步”
  • 在电脑打开 控制面板 - 程序 - 启用或关闭Windows功能 - Hyper-V，把勾选取消，确定后要重启电脑，在重新运行安装程序
    我的话是选择了第一种，因为我安装了Docker Desktop，必须要Hyper-V

用户体验设置这步默认是勾选“启动时检查产品更新”和“加入VMware客户体验提升计划”
启动时检查产品更新：每次打开VMware时，自动检测有没有新版本来提示你是否需要更新，就相当于是一个自动更新提醒（不会强制更新）
加入VMware客户体验提升计划：会把你在使用VMware的数据匿名发送给VMware用来改进软件
这两个勾不勾选没有影响，我这里是没有勾选
点击下一步

快捷方式，默认勾选“桌面”和“开始菜单程序文件夹”
桌面：在桌面添加快捷方式
开始菜单程序文件夹：在开始菜单中添加快捷方式

点击下一步

正在安装VMware

安装成功，点击完成即可

VMware的快捷方式会出现在你的桌面和开始菜单的文件夹中

检测虚拟网卡是否正常
使用快捷键按下“win+r”，输入“ncpa.cpl”，按回车就会打开网络连接窗口，只要显示了Vmware Network Adapter VMnet1和Vmware Network Adapter VMnet8说明下载正常，没有就证明安装有问题
Hyper-V Virtual Ethernet Adapter是因为我的Windows功能这里启用了Hyper-V，没有启用的就没有

双击桌面的VMware快捷方式，点击是

打开到如下页面

虚拟机安装rocky10.1过程

rocky官网链接：Rocky 官网
rocky镜像下载链接：rocky10.1镜像

打开虚拟机，选择新建虚拟机

选择典型，点击下一步

选择“稍后安装操作系统”，然后点击下一步

点击“Linux”，选择“Rocky Linux 64 位”

自定义虚拟机名称和安装路径

选择虚拟机的磁盘大小以及虚拟机的磁盘文件在你Windows（物理机）中时是以什么形式存在的
磁盘大小默认是20GB，根据个人需求设置大小（这个虚拟机磁盘大小不等于物理磁盘大小，不会占用实际磁盘空间）

  • 将虚拟磁盘存储为单个文件：VMware会在你的文件夹上创建一个巨大的.vmdk文件（D:\rocky10.1\rocky10.1.vmdk）
    优点：性能较好，在数据存储在物理磁盘上相对连续，读写效率更高
    缺点：难以迁移，拷贝几十甚至上百 GB 的单文件，对存储介质的文件系统格式（FAT32就可能不行）有严格要求
  • 将虚拟磁盘拆分成多个文件：VMware会把20GB磁盘切成多个约4GB作用的小文件存放在文件夹里（D:\rocky10.1\rocky10.1-s001.vmdk，-s002.vmdk以及类推）
    优点：方便移动和备份，想把虚拟机拷贝到 U 盘或移动硬盘，单个超大文件（比如 100GB）在旧格式磁盘（如 FAT32）上无法存储，且拷贝大文件容易出错。拆分成小文件更稳定
    缺点：性能较差，读写大量数据时，磁头或文件索引需要在多个小文件间切换，理论上性能比单文件稍差

根据个人需求选择，在这里我选将虚拟磁盘存储为单个文件，点击下一步

点击自定义硬件

内存和处理器根据需求来，我选择默认就行，然后点击“新 CD/DVD（IDE）” - “使用 ISO映像文件” - “浏览” 选择你镜像存放的位置

确认好信息之后就可以点击完成了

然后就可以开启虚拟机了，点击“开启此虚拟机”

进入系统安装引导界面，rocky10.1进入之后默认是选择第二个“Test this Media & install Rocky Linux 10.1”，点击上方向键选择“Install Rocky Linux 10.1”，按下回车
这两个区别不大，第一个更快一点
Install Rocky Linux 10.1：立即启动安装程序，跳过对安装介质（ISO 文件）的完整性检查
Test this Media & install Rocky Linux 10.1：在进入安装界面之前，系统会先对你挂载的 ISO 镜像进行一次全盘扫描，核对校验码（Checksum）

进入Linux系统后台服务的加载详情界面，启动（或关闭/重启）过程中都能看到，等待一下就可以跳到语言选择界面

如果选择的是第二个，它会出现进度条，可以使用esc键加速详情界面

到了下面这个界面就可以不用按esq键了

选择系统语言，根据个人需求来，我选择的是英语，然后点击Continue（继续）

点击“Installation Destination”（安装目标位置），就是把Linux安装到哪块硬盘，如何划分磁盘空间，就是在选择系统盘

默认磁盘是勾选上，直接点Done
Storage Configuration（存储配置），默认选择Automatic（自动），意思是系统会自动进行分区，适合新手

只要警告消失了就弄好了

选择Software Selection（软件选择）
默认就是Server with GUI（带图形界面的服务器），也就是图形化界面

Server with GUI（带图形界面的服务器）：有鼠标可以点，带桌面（通常是 GNOME），适合初学者
Server（服务器）：没有桌面，纯命令行，但预装了常用的服务器管理工具
Minimal Install（最小化安装）：最节省资源。 只安装最核心的运行组件，适合在追求极致性能、高安全性的生产环境，或者比赛中要求从零开始配置的情况
Workstation（工作站）：类似个人电脑，带有大量的办公软件、播放器等
Virtualization Host（虚拟化主机）：专门用来做母机，再在里面开 KVM 虚拟机的环境
我这里选择最小化安装，后面我会展示Server with GUI和Minimal Install和安装后的区别

点击“Root Account”（root账户）给Linux的root管理员设置密码
root管理员就是Linux中权限最大的一个用户，就好比做公司的老板，老板（root）有最高权限去管理公司（Linux系统）

点击“Enable root account”（启用root账户），并设置root管理员的密码
点击勾选“Allow root SSH login with password” （允许root用户使用密码进行SSH登录）

  • 如果勾选了的话，安装好Linux之后，可以windows上打开终端（CMD和PowerShell）使用ssh命令远程链接登录Linux（例如 ssh root@192.168.100.1）
  • 如果不勾选的话，要在Linux的ssh配置文件（/etc/ssh/sshd_config），修改第40行的“#PermitRootLogin prohibit-password”为“PermitRootLogin yes”，才可以使用终端使用ssh命令远程链接登录Linux

只要设置好root密码之后，USER SETTINGS（用户设置）就不会有警告了
下面的User Creation就是用来创建普通用户，根据个人需求，创不创建都可以。普通用户就没有root权限这么大，涉及到系统管理的就没有权限去执行，日常的命令权限还是可以的，就比可以比作为root手下的员工吧
点击“Begin Installation”（开始安装）

然后就等待虚拟机安装完毕就行

安装完毕之后，点击“Reboot System”（重启系统）

命令行的话就是一片黑色，然后有出现localhost login：的字样是让你输入账号的，输入完账号回车之后就是输入密码

图形化界面的话就类似与windows有开机动画到下面的页面点击“Start Setup”（开始设置）

Location Services是开启地理位置服务的，点击Next

这里就是创建一个普通用户的，名字随意，然后点Next

设置用户的密码，然后点Next

点击“Start Using Rocky Linux”（开始使用Rocky Linux）

点击win键，会有rocky10.1的引导教程，我这里就跳过了

点击“Terminal”（终端），也就是命令行了

默认当前用户是刚创建的那个用户，可以使用su –login命令切换用户，缩写是su -
