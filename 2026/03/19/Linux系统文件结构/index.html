Linux系统文件结构

xiao yu2026-03-192026-03-19

概念

文件系统 (File System) 可以拆解理解，“文件”顾名思义，主语就是文件，用于存储数据的基本单位，“系统”就是对这些文件进行组织和管理的机制，那通俗理解就是 “文件系统”是用来管理文件的一套规则和方法

文件系统是操作系统用于在存储介质上（如硬盘，U 盘等）管理数据的一种机制，不同的文件系统在 数据存储结构、访问速度、安全性、兼容性 等方面存在差异，因此被应用于不同的场景

常见的几个文件系统

  • Linux: ext2, ext3, ext4, xfsz
  • windows: NTFS, FAT32
  • U盘常见: exFAT
文件系统 适用系统 特点 适用场景
EXT4 Linux 高效稳定，日志功能 服务器、本地存储
NTFS Windows 高兼容性，支持权限控制 Windows 磁盘、外部存储
FAT32 Windows/macOS/Linux 兼容性强，但最大4GB文件限制 U 盘、存储卡
exFAT Windows/macOS 兼容性好，无4GB限制 大文件存储
XFS Linux 适合大文件，高速写入 高性能存储、数据库

感兴趣的可以自行去了解

目录结构

在Windows系统中，可以拥有多个磁盘分区（盘符），例如C盘，D盘，E盘等
在Linux系统中只存在一个根目录，目录结构类似一颗倒置的树，“/”是根目录，所有的文件目录都是存放在它下面，它是所有文件与目录的起点，是linux文件系统的顶层

目录详解

核心目录

目录 作用 常见文件/示例
/ 根目录，整个文件系统的起点，包含所有目录 无具体数据文件，只有子目录结构
/bin 基本命令（所有用户可用） ls, cp, mv, bash
/sbin 系统管理命令（root 使用） reboot, shutdown, fsck
/etc 系统配置文件目录 passwd, fstab, sshd_config
/home 普通用户家目录 /home/user1
/root root 用户家目录 /root/.bashrc
/boot 系统启动文件（内核、引导程序） vmlinuz, grub
/dev 设备文件目录（硬盘、终端等） /dev/sda, /dev/null
/proc 虚拟文件系统（进程、系统信息） cpuinfo, meminfo
/sys 内核与硬件信息 /sys/class
/lib 系统库文件（32位） *.so
/lib64 64位系统库文件 *.so
/usr 用户级程序和资源（最大目录之一） /usr/bin, /usr/lib
/var 可变数据（日志、缓存等） /var/log, /var/lib
/tmp 临时文件（重启可能清空） /tmp/test.txt

扩展目录

目录 作用 常见文件/示例
/opt 第三方软件安装目录 /opt/nginx
/mnt 临时挂载点 /mnt/data
/media 自动挂载设备（U盘、光盘） /media/usb
/run 系统运行时数据（内存中） /run/nginx.pid
/srv 服务提供的数据 /srv/www
/lost-found 文件系统修复数据 （自动生成）

/usr目录细分（重点）

目录 作用 常见文件/示例
/usr/bin 普通用户命令 python3,vim,git
/usr/sbin 系统管理命令 useradd，apache2
/usr/lib 库文件 各类 .so 动态库
/usr/local 本地安装软件 /usr/local/bin，/usr/local/lib

/var目录细分

目录 作用 常见文件/示例
/var/log 所有系统与服务日志的集中位置 syslog,auth.log,kern.log
/var/cache 应用和包管理缓存 /var/cache/apt
/var/lib 服务的持久化状态数据 mysql/,docker/

Linux tree命令

tree命令
作用：以树状结构显示指定目录的文件和文件夹结构

可以把根目录想象成一个文件柜，里面有很多抽屉（目录），第一层目录就是文件柜的第一层抽屉，例如/etc,/home,第二层就是抽屉里面的小分隔格，比如你拉开一个抽屉（/etc）,里面还有小格子（比如/etc/yum,/etc/shells）这些就是第二层
